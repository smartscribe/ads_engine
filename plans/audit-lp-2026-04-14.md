### 3. Adjacent features that stack the audit story (mention these in the body, don't make them the hero)

Source: same product_guide folder.

- **Note signing with audit checkpoints** (`documentation/notes.md`): Notes require signing before claim creation. Sign modal records modality, place of service, and attestation checkboxes (e.g. PDMP reviewed, telehealth consent). After signing, a SignatureBlock appears showing: "Electronically Signed" status, signer name, NPI, license number + state, license type, timestamp. Signed PDF includes all credential information. Amending a signed note moves it to "Amending" status, allowing re-edits before re-signing; original signed version remains accessible. **Why it matters for audit:** creates a legally defensible documentation trail and audit trail for payer review.
- **AI Note Editor — Audit tab** (`documentation/note-ai-editor.md`): Right-panel AI editor has three tabs — Edit, Audit, Analysis. The Audit tab surfaces compliance flags, questions, and scores from JotAudit with a badge showing count of open issues. Changes made through the AI editor go through an approval flow before final application. **Why it matters:** guides clinicians to edit notes in compliance-aware ways before signing.
- **Payer-optimized note templates** (`documentation/note-templates.md`): 32 templates in the public library, including specialty-specific and CPT-aligned templates (90832 / 90834 / 90837 / 90791 / 90792 / 99212–99215). Org admins can create custom templates and lock them to enforce consistency. Templates can include required fields for medical necessity, DSM-5-TR codes, Safety Assessment, etc. **Why it matters:** standardized, payer-aligned documentation structure reduces audit exposure before the note is even generated.
- **JotBill pre-submission audit review** (`billing/jotbill.md`): Before claim submission, JotBill shows a "Review JotAudit output (advisory) — green/yellow/red indicating documentation quality; does not block submission" and "Review scrubbing results — any ERROR-level scrub issues must be resolved before submitting." **Why it matters:** last-chance audit check before the claim goes to the payer.
- **JotCred provider credentialing** (`credentialing/jotcred.md`, beta): Intake app for payer enrollment. Providers upload required documents (license, DEA, board cert, NPI confirmation, W-9, malpractice insurance), answer 12 disclosure attestation questions, 3+ professional references, select payers. AI extracts structured data; providers verify + submit. Status Dashboard tracks payer application progress. **Why it matters for audit:** maintains audit trail of provider credentials and disclosures for payer files; supports recredentialing.
