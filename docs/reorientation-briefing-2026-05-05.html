2. Governing Thought

For three weeks, the ads were burning $200 to $475 per day against a measurement signal that did not exist. The fix was not a single bug. It was a stack of four bugs in Meta's undocumented Custom Conversion rule format, each one revealed only after the previous fix landed. We finally abandoned that approach Sunday night and routed the same intent through Meta's standard Purchase event template, which side-steps the entire failure surface. The plumbing is now built, verified, cleaned up, and inert. Turning the ads back on is the cheapest test of whether measurement was the actual constraint, or whether the April 16 campaign restructure also broke delivery.

3.1 Why the loop matters

The thing most people miss about Meta is that conversion events are not a reporting concept. They are the auction input. Meta's optimizer reads the dollar value attached to each Purchase event and decides whose feed to put your ad in next. When the signal was broken, the optimizer had nothing to weight against and was effectively running on impression-level proxies.4 When the signal is clean and value-weighted, Meta seeks out users statistically similar to the ones who have generated First Notes, the only event that maps to revenue. That is the loop the rebuild restored.

4. How We Got Here: The Three-Week Saga

The shortest version. Each fix landed clean code, surfaced a different undocumented Meta rule, then died. The fourth pivot abandoned Custom Conversions entirely. Today's signal flows through that fourth path.

Definitions. Custom Conversion (CC): a Meta object that says "if a pixel event matching rule X fires, count it as a conversion of type Y with dollar value Z." Standard event: a Meta-blessed event template (PageView, Purchase, Lead, etc.) that has known semantics across the entire ads network. Pixel rule: the JSON-encoded matcher that decides whether a given pixel fire qualifies for a CC. Learning phase: Meta's term for the first ~50 conversions on a new optimization target, during which delivery is unstable.

4.1 The lesson, written down so we do not repeat it

  • Meta's Custom Conversion API has at least four undocumented silent-failure modes. Whitespace, units, field name, multi-event VALUE incompatibility. Each one creates a working-looking object that never matches.6
  • Use Meta standard event templates for optimization signals. Custom Conversions are fine for reporting. They are a trap for optimization, because they sit between you and the auction's expectation of what the event "is."
  • Always verify with the canonical CC's last_fired_time field. Insights API has a 24-72 hour lag. The CC's last-fired timestamp updates within minutes and is the truth.

Appendix A. Glossary

Defined here in alphabetical order, in case a reference is wanted while reading.

TermDefinition
Ad setMeta's middle-tier object: lives inside a campaign, holds the targeting rules, the optimization goal, and the daily budget. Ads attach to ad sets.
Attribution windowHow long Meta waits after an ad click or impression before stopping the credit clock for a conversion. Ours: 7-day click + 1-day view.
CAPI (Conversions API)Meta's server-to-server event channel. The same event types as the browser pixel, but posted from a backend. Resilient to ad-blockers and iOS Safari Intelligent Tracking Prevention.
Custom Conversion (CC)A Meta object that says "if a pixel event matching rule X fires, count it as a conversion of type Y with dollar value Z." Most often built in the Meta UI; we used the API.
CompleteRegistrationA Meta standard event name. Mapped 1:1 to our Sign Up event in the dataLayer.
Cost per First Note (CpFN)Total ad spend divided by attributed First Note count. The single key performance indicator for the ads program.
dataLayerA single shared array on the page where the React app drops events. Tag managers (Google Tag Manager, GA4) listen to it.
EMQ (Event Match Quality)Meta's 0-10 score for how well a pixel event ties back to a real Meta user. Higher EMQ means better targeting and reporting. Browser-only events typically score 3-4; pairing with Conversions API and identity hashing typically scores 7+.
event_idA unique identifier on each conversion event, used by Meta to dedupe browser-side and server-side fires of the same action.
First Note (FN)Our canonical revenue-mapped event: a clinician completes their first AI-generated note. The only event that reliably correlates with becoming a paying customer.
fbqThe JavaScript function the Meta Pixel exposes for sending events from the browser.
GTM (Google Tag Manager)A hosted Google service that owns "if event X fires on page, send message Y to vendor Z." Decouples marketing-tag changes from app deploys.
Learning phaseMeta's term for the first ~50 conversions on a new optimization target, during which delivery is unstable. Exits to "Active" once enough signal accumulates.
Lifetime Value (LTV)Average total revenue a customer generates over their full relationship. Used to derive the dollar weight of First Note in the calibration step.
OUTCOME_SALESMeta's modern campaign objective for purchase-style events, replacing legacy OUTCOME_LEADS / OUTCOME_TRAFFIC objectives.
PixelA small JavaScript snippet running in the browser that sends a message to Meta whenever a tracked action happens. Nate's pixel: WebApp Actions, id 1625233994894344.
pixel_ruleA JSON-encoded matcher attached to a CC or an ad set's promoted_object. Decides whether a given pixel fire qualifies.
promoted_objectA field on a Meta ad set that says "this is what you are optimizing for." Holds the pixel ID, the event type, and any rule.
PurchaseA Meta standard event name. We are using it for all three of First Note, Sign Up, and Calendar Booking, distinguished only by content_name and value.
Sign Up (SU)A new clinician completes Auth0 registration on jotpsych.com. Mapped to Meta's CompleteRegistration standard event.
VALUE optimizationMeta optimization mode that bids based on the dollar amount on each conversion event, rather than count.
